# 원시데이터와 참조데이터 => 원시값, 참조값이라고도 부름
	+ 원시 데이터
		1. 기본 자료형(단순한 데이터)을 의미(Number, String, Boolean, Null, Undefined 등)
		2. 변수에 원시값을 저장하면 변수의 메모리 공간에 실제 데이터 값이 저장된다.
		3. 할당된 변수를 조작하려고 하면 저장된 실제 값이 조작된다.
		4. 원시값은 값을 복사 할 때 복사된 값을 다른 메모리에 할당 하기 때문에 원래의 값과 복사된 값이 서로에게 영향을 미치지 않는다.
		5. 원시 데이터 타입은 객체가 아닌 데이터 타입을 뜻한다.
			객체가 아닌 데이터가 무엇을 의미하냐면, 변수에 저장된 실제값에 직접적으로 접근할 수 있는 단순한 데이터라는 뜻이다.
		
	+ 참조 데이터 
		1. 여러 자료형으로 구성되는 메모리에 저장된 객체(Object, Array, function 등)
		2. 원시 데이터 타입이 아닌 모든 타입을 참조 데이터 타입이라고 한다.
		3. 변수에 객체를 저장하면 독립적인 메모리 공간에 값을 저장하고,
			변수에 저장된 메모리 공간의 참조(위치 값)를 저장하게 된다.
			그래서 할당된 변수를 조작하는 것은 사실 객체 자체를 조작하는 것이 아닌, 
			해당 객체의 참조를 조작하는 것이다.
			
	+ 원시 데이터와 참조 데이터의 차이
		1. 데이터의 변경 여부
			- 원시값은 변경 불가능
			- 참조값은 변경 가능
		2. 저장되는 값의 성격
			- 원시값이 변수에 할당된다면 실제값이 저장
			- 참조 자료형을 변수에 할당할 때는 변수에 값이 아닌 주소를 저장한다.
